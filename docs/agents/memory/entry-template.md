# Dreaming-Lite Entry Template

Copy this shape into [lessons.md](./lessons.md). Keep the title specific enough
that another agent can decide whether to read the entry.

```md
## YYYY-MM-DD - Short Lesson Title

Scope: runner | daemon | recovery | acceptance proof | publication | GitHub issues | prompts | tests | release | agent workflow
Evidence: command, issue, PR, test output, implementation spec, run summary, or linked doc
Lesson: One concise paragraph describing the repeatable pattern.
Use when: The task shape where this lesson should influence agent behavior.
Do not use when: Cases where this lesson would be misleading or too broad.
Review after: YYYY-MM-DD
```

Good entries describe a reusable failure mode or workflow correction. Bad
entries are raw transcripts, vague preferences, guesses, or one-off facts.
